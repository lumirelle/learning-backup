招聘管理系统（ATS）需求文档

版本：v1.1 状态：草稿 日期：2026-05-21 作者：AI 辅助调研 定位：全栈练手 / 考核押题 MVP

1. 项目背景

招聘管理系统（Applicant Tracking System，ATS）是 HR 团队管理岗位发布、候选人筛选与面试流程的核心工具。 本项目以 MVP 为目标，覆盖招聘全流程的核心链路，同时作为全栈开发学习的实战载体。

参考产品形态：Greenhouse、Lever、飞书招聘。

MVP 定位说明

本文档区分 MVP 与 后续迭代 两类需求，2周内仅交付 MVP 标注功能。

2. 目标与范围

2.1 业务目标

  • HR 可以在线发布、管理招聘岗位
  • 候选人可通过系统投递简历，查看申请状态
  • HR 可通过看板式界面直观管理候选人面试流程
  • 面试官可录入面试评价，形成评估记录
  • 管理员可查看招聘漏斗、效率等数据报表

2.2 MVP 范围（2周交付）

3. 角色与权限

角色标识核心权限
超级管理员 ADMIN 所有权限，包括用户管理、系统配置、查看全量数据报表
HR HR 岗位 CRUD、查看所有候选人、管理面试流程、录入面试评价
候选人 CANDIDATE 查看岗位列表、投递简历、查看自己的申请状态

4. 功能需求

4.1 用户认证模块

#用例角色描述
UC-01注册全部邮箱+密码注册，默认角色为 CANDIDATE；HR 账号由管理员后台创建
UC-02登录全部邮箱+密码登录，返回 JWT AccessToken（有效期 2h）+ RefreshToken（7d）
UC-03刷新 Token全部用 RefreshToken 换新 AccessToken，无需重新登录
UC-04修改密码全部需验证旧密码
UC-05登出全部前端清除 Token，服务端可选加入黑名单（Redis）

4.2 岗位管理模块

#用例角色描述
UC-10发布岗位HR填写岗位名称、部门、工作地点、薪资范围、JD 正文、招聘人数，状态默认"草稿"
UC-11编辑岗位HR可修改所有字段；已发布状态下修改需二次确认
UC-12变更状态HR草稿 → 开放 → 关闭；关闭后不再接受新申请
UC-13岗位列表HR / 候选人HR 可见全部；候选人仅可见"开放"状态岗位；支持关键字搜索、部门筛选
UC-14岗位详情全部查看完整 JD，候选人可在此页发起投递
UC-15删除岗位HR仅"草稿"状态可删除，已有申请的岗位不可删除

4.3 候选人管理模块

#用例角色描述
UC-20投递简历候选人填写姓名、手机、邮箱、工作年限、上传简历 PDF（≤5MB），一个岗位仅可投递一次
UC-21候选人列表HR按岗位筛选，支持搜索姓名/邮箱，显示当前阶段、投递时间
UC-22候选人详情HR基本信息、简历预览（PDF 内嵌查看）、流程历史时间线
UC-23查看投递状态候选人候选人登录后可看到自己的所有申请及当前阶段
UC-24标记不合适HR在任意阶段将候选人置为"已拒绝"，需填写原因

4.4 招聘看板（Pipeline）

核心亮点模块，看板按阶段展示所有候选人卡片，HR 可通过拖拽调整候选人所在阶段。

#用例角色描述
UC-30看板视图HR按岗位展示各阶段泳道，每张卡片含候选人姓名、头像缩写、投递时间
UC-31拖拽流转HR将候选人卡片从一个阶段拖入另一阶段，自动记录流转时间与操作人
UC-32快速操作HR卡片右键或悬停菜单：查看详情、标记拒绝、添加备注
UC-33阶段数量角标HR每列顶部显示该阶段候选人数量

招聘阶段定义（默认）

阶段英文标识说明

简历通过SCREENING_PASSHR 初步筛选通过

技术面试TECH_INTERVIEW专业技术轮次
HR 终面HR_INTERVIEWHR 综合面试
发放 OfferOFFER意向确定，待候选人确认

4.5 面试记录模块

#用例角色描述
UC-40添加面试评价HR选择面试轮次、填写面试官、评分（1-5星）、优劣势、最终结论（通过/拒绝/待定）
UC-41查看面试历史HR候选人详情页展示所有面试记录时间线，按轮次排序
UC-42编辑评价HR仅当日内可编辑

4.6 数据看板模块

#指标说明

7. API 设计

RESTful 风格，统一路径前缀 /api/v1，鉴权通过请求头 Authorization: Bearer <token>。

7.1 认证接口

POST/api/v1/auth/register用户注册Public
POST/api/v1/auth/login用户登录，返回 JWTPublic
POST/api/v1/auth/refresh刷新 AccessTokenRefreshToken
POST/api/v1/auth/logout登出JWT

7.2 岗位接口

GET/api/v1/jobs岗位列表（支持分页、筛选）JWT / Public
GET/api/v1/jobs/:id岗位详情JWT / Public
POST/api/v1/jobs创建岗位HR
PUT/api/v1/jobs/:id更新岗位HR
PATCH/api/v1/jobs/:id/status变更岗位状态HR
DELETE/api/v1/jobs/:id删除岗位（仅草稿）HR

7.3 申请（候选人）接口

POST/api/v1/jobs/:id/apply投递简历（含文件上传）CANDIDATE
GET/api/v1/applicationsHR：全量申请列表；候选人：自己的申请JWT
GET/api/v1/applications/:id申请详情JWT
PATCH/api/v1/applications/:id/stage变更候选人阶段（含拖拽）HR
PATCH/api/v1/applications/:id/reject拒绝候选人（含原因）HR

7.4 面试记录接口

GET/api/v1/applications/:id/interviews该申请的所有面试记录HR
POST/api/v1/applications/:id/interviews添加面试评价HR
PUT/api/v1/interviews/:id编辑评价（当日内）HR

7.5 数据看板接口

GET/api/v1/stats/funnel招聘漏斗（各阶段人数）HR
GET/api/v1/stats/overview月度概览数据HR

7.6 文件接口

POST/api/v1/files/upload上传文件（multipart/form-data），返回文件 URLJWT
GET/api/v1/files/:filename下载/预览文件HR
