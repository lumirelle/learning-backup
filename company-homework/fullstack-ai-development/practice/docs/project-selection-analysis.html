推荐方案详解

首选：招聘系统（ATS）

推荐
核心模块（MVP范围）
  • 岗位管理（JD发布、编辑、状态控制）
  • 候选人简历库（上传、解析展示）
  • 面试流程看板（拖拽卡片换阶段）
  • HR 与候选人双角色权限
  • 面试记录与评价
  • 数据看板（招聘漏斗统计）
技术亮点覆盖
  • 状态机设计（简历流转状态）
  • JWT 多角色权限控制
  • 文件上传与预览（PDF简历）
  • 拖拽看板（前端交互亮点）
  • PostgreSQL 关联查询设计
  • RESTful API 规范设计
为什么有亮点
产品素养 面试看板是业内标准交互（Greenhouse / Lever 产品形态），考官一眼认出，说明你理解了领域最佳实践。
UI / UX 看板拖拽 + 状态流转动画 + 候选人头像卡片，视觉表现力在5个系统里最强，且不依赖复杂的图表数据。
后端设计感 候选人状态流转是天然的状态机，展示出对业务领域建模的能力，而非简单 CRUD。

备选：培训学习系统（LMS）

候选

白板已标 ✓，说明公司可能已讨论过此方向。若考核题就是此系统，则优先选它练手。
核心亮点：课程卡片 UI（类 Coursera）、学习进度追踪、章节测评、证书生成。
技术覆盖：文件/视频存储、进度计算、问卷设计、多角色（管理员/讲师/学员）。

2周学习路径建议

阶段时间目标产出
环境 & 选型 Day 1–3 Java + PostgreSQL + Golang 基础扫盲，选定一种后端栈 环境搭建完成，跑通 Hello World API
读懂领域 Day 4–7 参考开源 ATS/LMS 项目，读懂数据模型和 API 设计 ERD 图 + API 文档草稿
核心开发 Day 8–11 借助 AI 从 0 写核心模块：认证、岗位/课程 CRUD、看板/进度 可运行的后端服务 + 前端主页面
打磨交付 Day 12–14 UI 打磨、联调接口、部署演示环境 可展示的 MVP + 演示视频
关于后端技术栈选择

建议以你已会的 Java（Spring Boot）快速起步，跑通全链路后再分支学 Golang。PostgreSQL 推荐直接上手，语法与 MySQL 差异不大，通过 AI 辅助可快速迁移经验。这样第一周不会卡在语言学习上，而是聚焦在架构和产品设计。
