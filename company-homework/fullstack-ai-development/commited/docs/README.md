## 二、文档规约

1. **`requirements.md` 只读**：作为不可变基线，任何变更只在本库其它文档中以「对需求的解读 / 扩展」形式体现。
2. **版本与时间**：每篇文档头部带元信息行（`版本 / 创建 / 最后编辑 / 状态 / 作者`），底部带「变更记录」表。每次实质性修改都需 bump 版本号并追加一条变更记录。
   - 版本号规则：`v<major>.<minor>`。`minor` 用于增量补充，`major` 用于结构性重写或评审定稿（v1.0 = 首次定稿）。
   - 状态流转：`草稿 → 评审中 → 已定稿（vX.0）→ 维护`。
3. **单一事实源（SSOT）**：数据结构以 `03-data-model.md` 为准，接口以 `04-api-design.md` 为准，二者冲突时以数据模型为基础对齐 API；产品口径以 `01-product-design.md` 为准。
4. **可追溯**：每个功能域在 `01` 有需求定义、在 `03` 有数据落点、在 `04` 有接口、在 `06` 有任务与验收，形成「需求→设计→实现→验收」闭环。
5. **命名**：文档用中文叙述，标识符（表名/字段/接口/枚举）用英文 `snake_case`，与代码保持一致。

## 四、关键事实速查

- **考核范围**：人力资源系统的「组织人事模块」，覆盖 10 个功能域（见 `01`）。
- **技术栈**：前端 Nuxt 3（Vue 3 + TS + Tailwind）｜ 后端 Go（Gin + Bun ORM）｜ 数据库 PostgreSQL 16 ｜ 缓存 Redis。
- **设计方向**：参考 `huashu-design` 技能产出高保真原型与设计语言，再翻译为生产级 Nuxt 组件（详见 `05`）。
- **交付物**：完整源码（含 README / DB 脚本 / 启动说明）+ 对外可访问的在线系统 URL（内网可达）。
- **关键时间**：考核 2026-06-05 启动，2026-06-10 截止，宽限至 2026-06-11。
