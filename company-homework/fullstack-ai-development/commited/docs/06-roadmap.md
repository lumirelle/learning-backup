## 1. 排期前提与现实校准

- **截止**：2026-06-10（今日），宽限至 2026-06-11。单人开发，时间极紧。
- **策略**：先用最短路径打通**一条端到端闭环（P0）**并完成部署，再向广度铺开（P1/P2）。**宁可少而完整，不要多而半成品。**
- 范围虽为「10 域全量铺开」，但落地按优先级**纵向切片**：P0 域做到可演示，P1 域打通主链路，P2 域给页面骨架 + 数据结构 + 至少一个可用动作。

## 5. 测试与质量

| 层次 | 做法 |
| --- | --- |
| 后端单测 | service 层关键逻辑（审批状态机、path 级联、数据权限拼接）用 Go test |
| 接口自测 | `.http`/Postman 集合或 `curl` 脚本覆盖主链路 |
| 前端 | 关键 composable/工具函数 vitest；页面以手动走查为主 |
| 端到端 | 一条「调动审批闭环」手动 E2E 验收脚本（见 §6） |
| 数据一致性 | 流程生效后校验 employees/job_history/hr_events 三处一致 |
| 静态检查 | Go：`go vet` + `golangci-lint`；前端：ESLint + `vue-tsc`/`nuxi typecheck` |

## 6. 验收清单（Demo 脚本）

1. 以 `super_admin` 登录 → 工作台显示指标卡与提醒。
2. 组织架构：新建部门、移动节点，架构图正确刷新。
3. 花名册：按部门+在职状态筛选 → 自定义列 → 导出 Excel；导入 50 行（含错误）看校验报告。
4. 打开某员工档案 → 发起「调动」流程 → 切换 `mgr01` 审批通过 → 回到档案看部门已变更、履历新增、动态时间线新增。
5. 人事动态/操作日志：检索到上述变更两条线记录。
6. 统计看板：切换组织范围，图表联动刷新。
7. 退出登录 → 以 `emp01` 登录只能看本人档案与发起本人流程。
